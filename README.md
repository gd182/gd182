## [ LORE ]

**Act I — Awakening**

It started the usual way: curiosity, a compiler, and too much free time. C++ was not the easy path. It never is. But it was the honest one — a language that doesn't hide what's happening beneath the surface. No garbage collector. No runtime to hold your hand. Just you, the hardware, and the consequences of your decisions.

**Act II — The Grind**

Five years. Fifty projects. One recurring lesson: *software rots when it's written for the deadline, not the use case.* I've seen codebases that nobody dares to touch. I've written some of them. Then I learned to write the other kind — the kind that still compiles two years later.

**Act III — The Multiclass**

At some point the domains started bleeding into each other. A Qt app needed firmware. The firmware fed a vision pipeline. The pipeline needed a dataset that didn't exist. So I built all of it. Not because I planned to be a generalist — but because the problems didn't respect domain boundaries, and neither did I.

**Philosophy**
```
1. Working software > clever abstractions
2. The best code doesn't need a comment to explain itself
3. If it doesn't compile on Linux, it's not done
4. Multithreading bugs are not bugs. They're features of your ignorance
5. Ship it. Then make it better
```
